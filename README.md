# AutoDev Framework

Фреймворк для автономной AI-разработки. Оркестрирует команду AI-агентов для непрерывной разработки программного обеспечения.

## Идея

Вместо того чтобы вручную управлять AI-ассистентами, AutoDev создаёт полноценную команду агентов с разными ролями, которые работают 24/7: берут задачи, пишут код, тестируют, ревьюят и деплоят.

## Агенты (роли)

| Роль | Задача | Триггеры |
|------|--------|----------|
| 🎯 PM | Создаёт задачи, приоритизирует | Расписание, ручные запросы |
| 👨‍💻 Developer | Пишет код, создаёт PR | Новая задача в очереди |
| 🧪 Tester | Тестирует через браузер как пользователь | PR создан, деплой на staging |
| 📊 BA | Оценивает UX/фичи | Деплой на staging |
| 📦 Release Manager | Собирает релизы, деплоит | N PR-ов готово |

## Ключевые принципы

1. **Event-driven** — агенты реагируют на события, а не поллят по расписанию
2. **Stateful** — очередь задач, статусы агентов, история решений
3. **Observable** — дашборд показывает что происходит прямо сейчас
4. **Configurable** — один YAML определяет проект, среды, агентов
5. **Provider-agnostic** — работает с Claude, GPT, Gemini, local models

## Стек

- **Python** — ядро оркестратора
- **FastAPI** — API + вебхуки
- **React** — дашборд
- **PostgreSQL** — состояние, очередь, история
- **Playwright** — браузерное тестирование
- **Docker** — деплой

## Статус

🚧 В разработке. Прототип работает на базе OpenClaw cron jobs.

## Предыстория

Этот проект родился из практического опыта построения автономной разработки для [Great Alerter](https://github.com/zinchenkomig/great_alerter_backend) — системы алертинга метрик. За 5 дней команда AI-агентов создала 30+ PR-ов, но обнаружились ограничения подхода на cron jobs и текстовых файлах.
